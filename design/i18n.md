# 多言語対応 (言語パック方式)

SelfMatrix は「日本語を直書きせず、言語パック (locale JSON) を足すだけで対応言語が増える」設計。
2026-07-04 に英語 (基底) + 日本語 (第 1 パック) + 言語セレクタを実装。

## 仕組み

言語設定は 1 つ (Settings > General の Language、localStorage 永続) で、cinny の文言と
通話 UI (Element Call) の両方を同じ言語に揃える。

### cinny (selfmatrix-cinny)

- 既存の i18next 基盤 (`src/app/i18n.ts`、`public/locales/<lng>.json` を読む) を活用。
- **branding 名前空間**: Discord 風の呼称 (サーバー/チャンネル) を `public/locales/en.json` の
  `branding.*` に置き、`useBranding()` フック (`src/app/branding/strings.ts`) が `t()` 経由で読む。
  `public/locales/ja.json` が日本語パック。
- 言語セレクタ: `src/app/hooks/useLanguageItems.ts` + General.tsx。値は `settings.language`
  (`'system' | 'en' | 'ja'`)。`src/app/hooks/useLocaleSync.ts` が `i18n.changeLanguage` に反映
  (system 時は navigator 言語を supportedLngs と照合)。
- `src/app/i18n.ts` の `supportedLngs: ['en', 'de', 'ja']`。
- **通話 widget への受け渡し**: `CallEmbed.ts` が `lang: i18n.resolvedLanguage ?? 'en'` を
  widget URL に付与 (旧: `'en-EN'` 固定)。

### element-call (selfmatrix-element-call)

- upstream が既に言語パック構造 (`locales/<lng>/app.json`) で、**ja を含む多数の言語を同梱**。
- widget モードは URL の `lang` パラメータで言語決定 (`urlFragment` detector)。cinny から渡る。
- SelfMatrix が追加した通話 UI キー (ピン/視聴ゲート/分割/ミニタイル位置) は `locales/en/app.json`
  に加えて `locales/ja/app.json` にも翻訳を追加済み。それ以外の本体文言は upstream の ja がそのまま効く。

## 言語を足す手順 (例: フランス語)

1. cinny: `public/locales/fr.json` を作り、`branding` 名前空間を訳す (en.json をコピーして翻訳)。
   `i18n.ts` の supportedLngs に `'fr'` を追加、`useLanguageItems.ts` のセレクタに 1 行追加。
2. element-call: upstream に `locales/fr/app.json` があればそのまま効く。SelfMatrix 追加キー
   (video_tile.pin など 8 個 + settings.preferences_tab.mini_tile_strip_position 系 6 個) の
   fr 訳を `locales/fr/app.json` に追加 (無ければ英語フォールバック)。
3. コード変更は基本的に不要 (セレクタへの 1 行と supportedLngs のみ)。

## スコープと今後

- **完了**: branding 用語の日本語化 + 通話 UI の日本語化 (パック方式の枠組み)。
- **未対応 (別途の大きな作業)**: cinny 本体の全 UI (ログイン・設定・ルーム・メッセージ編集など) は
  文字列が JSX 直書きの英語で i18n 化されておらず、日本語化するには各コンポーネントを `t()` 化して
  `ja.json` に全訳を足す漸進作業が必要。フレームワークは整ったので、アーキテクチャの作り直しは不要。

v1.6 では 2026-07-14 の実機比較とドッグフーディングを反映する: native のメイン/別窓を同じ Element Call
フッターへ統一する、通話参加/終了イベントを表示しない、自動追従 PiP は作らない、明示的な配信単体

v1.7 ではカメラを再導入する。ただし配信中心の既定 UI は維持し、機能設定は既定 OFF、設定 ON と
各通話での明示 ON の二段階を必須にする。

## 基本方針

- **配信中心・カメラは明示 opt-in**: 既定は音声 + 画面共有(配信)のみ。カメラ機能は全体設定で既定 OFF とし、
  ON にした時だけ参加前/通話中のカメラ操作とビデオ設定を表示する。設定 ON だけでは送信せず、その通話で
  ユーザーが明示的にカメラを ON にした場合だけ capture/publish する
- **カメラ状態を持ち越さない**: 保存済みのカメラ ON は起動時に OFF へ正規化する。参加前の ON 選択は
  その 1 回の参加だけに適用し、次回参加では OFF に戻す。通話中は feature 設定自体を変更不可にする
- **権限要求を操作意図に限定**: 機能 OFF 時は EC の video capability も無効化し、ビデオ設定タブを隠す。
  web iframe の Permissions Policy からも `camera` を外す。音声設定や音声デバイスメニューからカメラ
  権限を要求せず、カメラのデバイス名取得はビデオ設定を明示的に開いた時だけ行う
- **配信の視聴はオプトイン**: 強制参加(自動再生)ではなく、見るかどうかを視聴者が選ぶ。
  見ていない配信は購読しない(帯域面でも合理的 — simulcast 購読自体を止める)
- **リアクション・効果音系は初期スコープ外**: Discord に相当機能はあるが、通話中の視界と音量安全性を優先し、
  SelfMatrix では非表示または無効化する。代わりにユーザー単位・配信単位の音量調整は必須機能として扱う
- **自動追従 PiP は不採用**: チャット等へ移動した際にアプリ内ミニプレイヤーを自動表示しない。通話全体の
  明示 popout と、将来の配信単体の明示 popout は別機能として扱う

## 通話画面(element-call fork 側)

- **表示モードは注視⇔グリッドの 2 モード切替式**(v1.3 で 3 段構えから改訂):
  1. **注視(基本・既定)**: Discord 準拠。1配信を大きく表示 + 他はミニタイル。
     **ミニタイルをクリックすると注視対象がそのタイルに切り替わる**(v1.4)。
     ただし Cinny 埋め込みの通常画面では、視聴中の配信が 1 つでもある場合は話者ミニタイルを出さない。
     話者状態は配信上のオーバーレイで見せる
  2. **グリッド**: 視聴中の配信を**正方形に詰めて**均等表示(2本=1×2、3〜4本=2×2、
     5〜9本=3×3 …)。横一列に並べる方式ではない
- **視聴をオプトインした配信だけがタイルになる**: 未視聴の配信はタイルを作らず
  画面を分割しない。「視聴できる配信」の導線(チップ等)から視聴を開始する
- **グリッド中の「強調選択」(ON/OFF)**: ON にしてタイルを選ぶと、**選択したタイルだけで
  グリッドを組み直し(正方形詰めのまま)、選択しなかったものがミニタイルになる**。
  選択数は任意(**1 枚でも 3 枚以上でも可**)。OFF に戻せば均等グリッドへ。
  あくまで「グリッドの表示対象を絞る」操作であり、**モードはグリッドのまま**(v1.4)
- **強調選択で 1 枚だけ選んだときは見た目が注視モードと同じになるだけ**で、
  注視モードへの自動遷移はしない(v1.4 で改訂 — モード切替は常にユーザーの明示操作)
- **視聴中の配信が 1 つでもあれば話者ミニタイルを出さない**。メイン埋め込み/別ウィンドウのどちらでも
  同じ規則とし、話者状態とユーザー音量への導線は配信上の話者オーバーレイへ集約する
- **ミニタイルの位置はユーザー選択可**: 上・下・左・右
- **話者表示は Discord StreamKit Overlay 風**: アバター+名前ピルを配信画面にオーバーレイ。
  発話中は緑リング+名前ハイライト、ミュートはマイクオフアイコン。
  **オーバーレイはドラッグで移動可能**(これのみ直接操作、他の配置設定は設定画面)。
  オーバーレイまたは話者ピルの右クリックから、そのユーザーのミュート・音量調整へ到達できる (実装済み)
- **受信品質を見える化する**: 視聴中の配信には 720p 30 FPS などの品質バッジを表示し、カクつきが
  送信側・受信側・設定のどこに起因しそうか判断できる材料を出す
- **ポップアウトの主対象は「通話 UI 全体」**(v1.3 で再昇格): 通話 UI を丸ごと別ウィンドウへ
  出す。**目的は通話 UI とチャット UI を同時に見て・触ること**。native は同じ
  `WebContentsView`を再親子付けするため再接続せず、別窓を閉じるとメインへ戻って通話を維持する
- 配信ストリーム単位の明示ポップアウト(タイルの 🗗、再接続なし)はweb/EC版を実装済み。nativeは
  rendererの`window.open`を拒否するためhost連携が未実装で、動かないボタンは非表示にする。実装時も
  ユーザー操作で開始し、画面遷移に自動追従する PiP にはしない
- **配信の設定(画質と FPS)は配信者が選べる — Discord 準拠**(v1.4 追加):
  画面共有の開始時に解像度(720p/1080p/ソース解像度)と FPS(15/30/60)をピッカーで
  選択できる。既定はソース解像度/60fps。requirements.md §3 に対応要件あり
  (Sonnet レビュー M5 の「選択 UI なしは意図的差別化」判断はこの合意で上書き)
- **通話ステータス文言**: 画面共有が 0 本の通話を「ライブ配信中」とは表示しない。
  通話中は `通話中 {{count}}人`、配信中は配信者/配信数が分かる文言へ分ける
- **カメラ操作**: `設定 > 全般 > 通話 > カメラ機能を有効にする` が OFF の間は参加前/通話中の
  カメラボタンを表示しない。ON にしてもカメラの初期状態は OFF とし、参加前または通話中のボタン操作を
  publish の同意として扱う。native では Element Call 共通フッター、web では Cinny 通話バーから操作する
- **native の窓挙動**: 通話はメインで開始し、明示操作で通話全体を別窓へ移す。メイン/別窓とも同じ
  Element Call フッターを使い、別窓を閉じる = メイン復帰・通話継続。開き方の設定や二層保存は作らない。
  [native-milestones.md](../planning/native-milestones.md) M3 と [m3-window-ux.md](m3-window-ux.md) を正とする
- **タイムライン**: Discord と同様、通話参加/終了イベントをメッセージ履歴へ表示しない。通話状態は
  ボイスチャンネル参加者、通話画面、ステータス領域で示す

## シェル(cinny fork 側)

- **サーバー一覧枠・チャンネル一覧枠とも 上・下・左・右 から選択可**
  (上下配置時は横並び UI。チャンネル一覧はチップ列)
- **配置の設定 UI は設定画面に置く**(ドラッグハンドル方式にはしない)
- **設定画面などの PageNav は横チップ化しない**: サーバー/チャンネルリストの上下配置は main route の
  明示 opt-in に限定する。設定モーダルや管理系画面の縦ナビを壊さない
- **上下配置時の横チップ nav はカテゴリ内アイテムも横並び対応**: チャンネルリストを右/上/下に動かしても
  通話画面やチャンネル一覧が見切れないことをレイアウト要件に含める

## デフォルト(初見の友達が見る形)

- **配置セットアップの案内は SelfMatrix の初回利用時に 1 回だけ**(v1.3 で明確化)。
  まず Discord 風 / カスタムを選ばせ、カスタム時だけ詳細な配置を選ぶ。通話参加時やルーム参加時に
  毎回は出さない。あとから設定画面でいつでも変更できる旨を添える
- スキップ/未選択時のフォールバックは **Discord 風**: サーバー枠・チャンネル枠は左、
  ミニタイルは下、表示モードは注視(Discord と同じ挙動線)
- 通話の開始位置設定は作らない。native はメインで開始し、必要な時だけ明示的に別窓へ移す

## 実装マッピング

| 対象 | fork | 対象コード(スパイクで特定済みの周辺) |
|---|---|---|
| 表示モード・注視選択・ミニタイル制御・話者オーバーレイ | element-call | SpotlightTile / grid 系 UI(src/tile/, src/grid/) |
| カメラ feature gate・既定 OFF・参加時同意 | cinny + element-call | Cinny settings/call preferences/widget URL、EC `disableVideo`/device settings |
| 配信オプトイン視聴(購読制御) | element-call | LiveKit track subscription 制御 |
| メディアパラメータ(高画質化) | element-call | src/livekit/options.ts(スパイク項目2で実証済み) |
| サーバー枠・チャンネル枠のドッキング | cinny | ClientLayout 系。DOM 構造は upstream のまま CSS(flex-direction / order)+設定切替で実現 |
| native 通話画面ポップアウト | desktop + element-call | WebContentsView を無再接続で再親子付け。EC 共通フッターの狭い preload API から popout/popin/pin/fullscreen を操作 |
| 配置設定の保存 | cinny | Matrix account data(端末間同期が効く) |

## リスク・注意

- ドッキング構想は fork-strategy.md「差分は小さく」と緊張関係 →
  レイアウト層の追加 + CSS 切替で実装し、upstream 追従の衝突面を最小化する
- EC のタイル UI は upstream でも活発に変わる箇所 → 新レイアウトは別コンポーネント追加で
  既存を温存する
- native の無再接続再親子付けは実装・E2E/probe 済み。残る重点確認は実アカウント2名での画面共有、
  共有音声、メイン/別窓の実機見た目
- 初回セットアップの促しは「うるさくない」形にする(モーダル連発にしない)
